# rigor_ledger.txt — the rigor ratchet's committed state
# =======================================================
#
# This file records aggregate RIGOR METRICS of the codebase as committed state.
# The regression test `tests/regressions/rigor_ratchet.rs`
# (`rigor_does_not_silently_regress`) recomputes each metric from the working
# tree and asserts that strictness is MONOTONE over time:
#
#   * floor metrics may NOT decrease below the value recorded here, and
#   * the nan_emissions CEILING may NOT increase above the value recorded here.
#
# WHY: a hack that removes a guard, deletes an assertion, launders an error into
# a silent fallback, or drops a build.rs ban gate necessarily DROPS one of these
# metrics — and adding a NaN emission raises the ceiling. The ratchet turns any
# such silent weakening into a hard test failure. The ONLY way to lower a floor
# (or raise the ceiling) is to edit this ledger in the same commit. That edit is
# visible in the diff, so a reviewer sees e.g. `panic_guards = 194 -> 192` and
# asks why, instead of the weakening hiding inside a "graceful fix". Strictness
# can therefore only drop deliberately and visibly, never silently.
#
# HOW THE METRICS ARE COMPUTED (must match the test exactly):
#   - Scope: the NON-test `src/` tree only. Lines inside test scopes
#     (`#[cfg(test)]`-gated blocks and `mod tests {` / `mod test {` blocks) are
#     excluded via a brace-depth state machine. `tests/` is not counted.
#   - panic_guards   = occurrences of `panic!(`                          (floor)
#   - asserts        = `assert!(` + `assert_eq!(` + `assert_ne!(`        (floor)
#   - result_guards  = occurrences of `return Err(`                      (floor)
#   - ban_gates      = `fn scan_for_` in build.rs
#                      + `forbid_build_rs_self_tampering` in build.rs     (floor)
#   - nan_emissions  = occurrences of `f64::NAN` in non-test src/  (CEILING:
#                      a non-increase bound, NOT a floor — must not go UP)
#
# Seeded from origin/main; these are the EXACT current counts, so the test
# passes clean on main. To change a value you must justify the change in the
# commit that edits this line.
