# sites. Auto-managed by build.rs — do NOT hand-edit. Each non-comment
# line is tab-separated:
#   <relative_path>\t<normalized_fn_signature>\t<comma_marker_kinds>
# When an entry disappears from the source tree, build.rs inspects the
# enclosing function's body. Trivial bodies (under a few code lines)
# and bodies that still contain any panic-shape macro fail the build,
# so deleting a `unimplemented!` without writing a real implementation
# is caught. Legitimately-implemented removals auto-prune.
#
# Concurrent branches that both remove markers will produce additive
# line-level diff conflicts that resolve by union — the next build
# then re-validates each surviving entry.
